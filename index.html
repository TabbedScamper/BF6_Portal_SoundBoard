How these sounds are sourced

Every sound here comes straight from the game's own asset list — nothing hand-picked. Battlefield 6 Portal ships a generated SDK whose RuntimeSpawn_Common enum lists every spawnable asset; the entries beginning with SFX_ are the sound effects the PlaySound function can play.

An in-game tool reads that enum at runtime (Object.keys(mod.RuntimeSpawn_Common)), keeps the SFX_ ones, and plays each at an isolated point while the audio is recorded — then the recording is split back into one clip per sound by the game-time it played.

Because the list is read directly from the SDK enum (which DICE auto-generates from the game build), it is the complete, authoritative set for this version. Portal SDK v? lists ? capturable SFX across ? categories — all captured. (The SDK has one extra SFX entry, SFX_VOModule, which is a silent carrier object PlayVO uses — it has no audio of its own, so it isn't listed.)

Announcer voice-overs are separate from the SFX list: 61 game-action callouts triggered with PlayVO. On the live build the announcer fires the wrong line or stays silent at random (a known, still-open engine bug), so only a handful actually play. The ones that do are kept; the rest are listed as silent placeholders. They're the Announcer category.

Crash Sounds are the few assets that crash the game when played. Their audio (captured right up to the crash) is kept and flagged in red so you don't trigger them in a mod.

Anything that produces no audio in-game — silent assets and unreliable VO — still appears for completeness, flagged with yellow caution tape; game-crashers get red.
